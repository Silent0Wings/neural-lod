@startuml Stage_1_Evaluation_Metrics

title Stage 1: Evaluation & Metrics Analysis (Activity)

start

:EvaluationLogger or
compare_train_vs_eval.ipynb
Loads eval_neural_*.csv;

:Parse CSV columns:
- elapsed_s (time)
- cpu_frame_ms
- gpu_frame_ms
- lod_bias (predicted)
- over_budget flag;

:Compute Performance Statistics;

  :Calculate mean, median,
  P95, P99, max CPU time;

  :Count frames over
  TARGET_FRAME_MS (16.66ms);

  :Percentage over-budget =
  frames_over / total_frames;

  :Mean LOD bias value;
  :Std dev of bias;
  :Distribution histogram;

  :Calculate flip_rate_hz =
  bias_switches / elapsed_seconds;

  if (flip_rate_hz < 1.0 Hz) then (yes)
    :✓ STABLE (acceptable jitter)
  else
    :⚠ JITTERY (needs filter)
  endif

  :Angular delta between frames
  (to detect flicker);

  :Correlation - bias vs velocity
  (should be positive);

  :GPU Savings (ms) =
  aggressive_gpu_time - current_gpu_time;

  :Inference Cost (ms) =
  model_forward_pass_duration;

  :Net Gain = GPU_Savings - Inference_Cost;

  if (Net Gain > 0.5 ms) then (yes)
    :✓ PROFITABLE (worth using)
  else
    :⚠ BREAK-EVEN or LOSS
    (model too slow or bias not aggressive)
  endif

:Output Comparison Plots;

note right
  compare_eval_vs_train.png
  Contains 9 subplots:
  - CPU/GPU latency curves
  - Bias time-series
  - Bias distribution
  - Over-budget % bar chart
  - Flip rate frequency
  - Correlation heatmaps
end note

:Print Summary Report;

note right

  ──────────────────

  Flip rate: 0.8 Hz ✓
  Net gain: +0.52 ms ✓
  ──────────────────
end note

stop
